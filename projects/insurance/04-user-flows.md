## Summary
Mobiele flow om binnen minuten verzekerd te zijn.

## Actor
Nieuwe gebruiker (18–35)

## Preconditions
- Smartphone
- Geldige ID
- Bankrekening

## Primary Flow (happy path)
1. App openen
2. Verzekering kiezen
3. ID-scan + selfie
4. Automatische datavulling
5. Premie tonen
6. 1-tap bevestigen (Apple Pay/iDEAL)
7. Directe polis in app

## Alternate flows
- ID verificatie mislukt → retry
- Extra risico → korte aanvullende vraag
- Betaling mislukt → alternatieve methode

## Postconditions
Actieve polis + digitaal bewijs.

## Success criteria (example metrics)
- <3 min afrondtijd
- <10% drop-off
- >80% volledig automatisch geaccepteerd

## UI copy examples (short)
- “Klaar in 2 minuten.”
- “Wij vullen dit voor je in.”
- “Je bent verzekerd.”

# User Flows – Claim indienen :contentReference[oaicite:4]{index=4}

## Summary
Schade melden via foto met automatische beoordeling.

## Actor
Bestaande klant

## Preconditions
Actieve polis.

## Primary Flow (happy path)
1. “Meld schade”
2. Foto/video upload
3. AI-inschatting
4. Bevestiging bedrag
5. Directe uitbetaling

## Alternate flows
- Fraude-detectie → handmatige review
- Onvoldoende bewijs → extra upload
- Buiten dekking → duidelijke uitleg

## Postconditions
Claim afgerond of in review.

## Success criteria (example metrics)
- 70% straight-through processing
- <24u uitbetaling
- Claim NPS >50

## UI copy examples (short)
- “Maak een foto.”
- “We regelen dit direct.”
- “Geld is onderweg.”# Summary
