## 1. Core Principles

### Conversation First
All interaction happens via natural language.
Prevents screen-heavy interfaces.
Violation: adding multi-tab dashboards.

### Action Over Insight
We deliver tasks, not charts.
Prevents analysis paralysis.
Violation: introducing KPI dashboards.

### Automation by Default
Data is captured automatically.
Prevents manual CRM updates.
Violation: required field forms.

---

## 2. Product Boundaries (What We Are Not)

- Not a reporting tool
- Not a BI dashboard
- Not a data-entry system
- Not customizable with complex layouts

---

## 3. Core Product Behaviours

- Proactively suggests actions
- Explains priority briefly when asked
- Updates records invisibly
- Minimizes user choices

---

## 4. Decision-Making Rules

- Remove screens before adding features
- Prefer execution over visibility
- Reduce input fields
- If it doesn’t lead to action, don’t build it

---

## 5. Product Character

Should feel:
- Focused
- Direct
- Calm
- Decisive

Should not feel:
- Analytical
- Busy
- Gamified
- Over-configurable

---

## 6. Language & Tone

- Direct and minimal
- No motivational language
- No buzzwords
- Action-oriented

Example:
Instead of: “Pipeline health improved.”
Use: “Call 3 deals at risk.”
